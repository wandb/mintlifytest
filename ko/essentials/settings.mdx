- title: 전역 설정
- description: Mintlify는 docs.json 파일을 사용하여 문서의 모양과 느낌을 완전히 제어할 수 있게 해줍니다
- icon: gear
---

모든 Mintlify 사이트는 `docs.json` 파일이 필요하며 이 파일에는 핵심 구성 설정이 포함되어 있습니다. [properties](#properties)에 대해 아래에서 자세히 알아보세요.

## 속성

<ResponseField name="name" type="string" required>
  프로젝트 이름입니다. 전역 제목으로 사용됩니다.

  Example: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  그룹 내의 모든 페이지가 포함된 그룹 배열

  <Expandable title="네비게이션">
    <ResponseField name="group" type="string">
      그룹의 이름입니다.

      Example: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      페이지 역할을 하는 마크다운 파일의 상대 경로입니다.

      Example: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  로고 이미지 경로 또는 "light"와 "dark" 모드 로고 이미지 경로가 있는 객체

  <Expandable title="로고">
    <ResponseField name="light" type="string">
      라이트 모드에서의 로고 경로
    </ResponseField>

    <ResponseField name="dark" type="string">
      다크 모드에서의 로고 경로
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      로고를 클릭했을 때 연결되는 링크
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  파비콘 이미지 경로
</ResponseField>

<ResponseField name="colors" type="Colors">
  전역 테마를 위한 16진수 색상 코드

  <Expandable title="색상">
    <ResponseField name="primary" type="string" required>
      기본 색상입니다. 라이트 모드에서 강조된 콘텐츠, 섹션 헤더, 액센트 등에 가장 자주 사용됩니다
    </ResponseField>

    <ResponseField name="light" type="string">
      다크 모드용 기본 색상입니다. 다크 모드에서 강조된 콘텐츠, 섹션 헤더, 액센트 등에 가장 자주 사용됩니다
    </ResponseField>

    <ResponseField name="dark" type="string">
      중요한 버튼을 위한 기본 색상
    </ResponseField>

    <ResponseField name="background" type="object">
      라이트 모드와 다크 모드 모두에서의 배경 색상

      <Expandable title="객체">
        <ResponseField name="light" type="string" required>
          라이트 모드에서의 배경 16진수 색상 코드
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          다크 모드에서의 배경 16진수 색상 코드
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  배열의 `name`와 `url`는 상단 바에 포함하려는 링크입니다

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      버튼의 이름입니다.

      Example: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      버튼을 클릭했을 때의 URL입니다. 예: `https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="상단 바 Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      링크는 버튼을 표시합니다. GitHub는 제공된 URL의 저장소 정보와 GitHub 스타 수를 표시합니다.
    </ResponseField>

    <ResponseField name="url" type="string">
      만약 `link`: 버튼이 연결되는 링크입니다.

      만약 `github`: GitHub 정보를 불러올 저장소 링크입니다.
    </ResponseField>

    <ResponseField name="name" type="string">
      버튼 내부의 텍스트입니다. `type`가 `link`인 경우에만 필요합니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  버전 이름의 배열입니다. 네비게이션 바에 드롭다운으로 다른 문서 버전을 표시하려는 경우에만 사용하세요.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  앵커의 배열로, `icon`, `color`, 그리고 `url`를 포함합니다.

  <Expandable title="앵커">
    <ResponseField name="icon" type="string">
      앵커를 특징짓는 데 사용되는 [Font Awesome](https://fontawesome.com/search?q=heart) 아이콘입니다.

      Example: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      앵커 레이블의 이름입니다.

      Example: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      어떤 페이지가 앵커에 속하는지 표시하는 URL의 시작 부분입니다. 일반적으로 페이지를 넣은 폴더의 이름입니다.
    </ResponseField>

    <ResponseField name="color" type="string">
      앵커 아이콘 배경의 16진수 색상입니다. `from`와 `to` 속성이 각각 16진수 색상인 객체를 전달하면 그라데이션으로 설정할 수도 있습니다.
    </ResponseField>

    <ResponseField name="version" type="string">
      올바른 문서 버전이 선택될 때까지 앵커를 숨기려는 경우 사용됩니다.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      직접 링크를 통해 앵커 내부의 문서로 연결할 때까지 앵커를 숨기려면 `true`를 전달하세요.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      다음 중 하나: "brands", "duotone", "light", "sharp-solid", "solid", 또는 "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  최상위 앵커의 기본 구성을 재정의합니다.

  <Expandable title="객체">
    <ResponseField name="name" default="Documentation" type="string">
      최상위 앵커의 이름
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome 아이콘.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      다음 중 하나: "brands", "duotone", "light", "sharp-solid", "solid", 또는 "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  네비게이션 탭의 배열입니다.

  <Expandable title="탭">
    <ResponseField name="name" type="string">
      탭 레이블의 이름입니다.
    </ResponseField>

    <ResponseField name="url" type="string">
      어떤 페이지가 탭에 속하는지 표시하는 URL의 시작 부분입니다. 일반적으로 페이지를 넣은 폴더의 이름입니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 설정을 위한 구성입니다. API 페이지에 대해 자세히 알아보려면 [API Components](/api-playground/demo)를 참조하세요.

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      모든 API 엔드포인트의 기본 URL입니다. `baseUrl`가 배열인 경우, 사용자가 전환할 수 있는 여러 기본 URL 옵션을 활성화합니다.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="인증">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          모든 API 엔드포인트에 사용되는 인증 전략입니다.
        </ResponseField>

        <ResponseField name="name" type="string">
          API 플레이그라운드에서 사용되는 인증 매개변수의 이름입니다.

          메서드가 `basic`인 경우, 형식은 `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          인증 입력 필드의 접두사로 설계된 기본값입니다.

          예를 들어, `inputPrefix`가 `AuthKey`인 경우 인증 필드의 기본 입력 결과는 `AuthKey`가 됩니다.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API 플레이그라운드 구성

      <Expandable title="플레이그라운드">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          플레이그라운드가 표시되는지, 숨겨져 있는지, 또는 사용자 상호작용 없이 엔드포인트만 표시하는지 여부 `simple`

          자세한 내용은 [playground guides](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      이 플래그를 활성화하면 OpenAPI 페이지의 키 순서가 OpenAPI 파일에 정의된 키 순서와 일치하도록 보장합니다.

      <Warning>이 동작은 곧 기본적으로 활성화될 예정이며, 그 시점에서 이 필드는 더 이상 사용되지 않을 것입니다.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI 파일을 가리키는 URL 또는 상대 경로의 문자열 또는 문자열 배열입니다.

  Examples:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  키:속성 쌍이 소셜 미디어 플랫폼과 계정 URL을 나타내는 소셜 미디어 계정 객체입니다.

  Example:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      다음 값 중 하나 `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Example: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      소셜 플랫폼의 URL입니다.

      Example: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  피드백 버튼을 활성화하기 위한 구성

  <Expandable title="피드백">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      사용자가 풀 리퀘스트를 통해 편집을 제안할 수 있는 버튼을 활성화합니다
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      사용자가 문서에 대한 이슈를 제기할 수 있는 버튼을 활성화합니다
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  다크 모드 토글을 사용자 정의합니다.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      새 사용자에게 항상 라이트 모드 또는 다크 모드를 표시할지 설정합니다. 설정하지 않으면 사용자의 운영 체제와 동일한 모드로 기본 설정됩니다.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      다크/라이트 모드 토글을 숨기려면 true로 설정하세요. `isHidden`를 `default`와 함께 사용하여 문서가 라이트 모드나 다크 모드만 사용하도록 강제할 수 있습니다. 예를 들면:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  모든 페이지 뒤에 표시될 배경 이미지입니다. 예시는 다음을 참조하세요:

  [Infisical](https://infisical.com/docs)과 [FRPC](https://frpc.io).
</ResponseField>
