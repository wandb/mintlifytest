もし専用クラウドインスタンスで管理されているすべてのデータをエクスポートしたい場合、W&B SDK APIを使用して、[インポートおよびエクスポートAPI](/ja/ref/python/public-api/)を利用して runs、メトリクス、Artifacts などを抽出できます。以下の表は、いくつかの主要なエクスポートユースケースについて説明しています。

| 目的 | ドキュメント |
|---------|---------------|
| プロジェクトのメタデータをエクスポート | [Projects API](/ja/ref/python/public-api/projects/) |
| プロジェクト内の runs をエクスポート | [Runs API](/ja/ref/python/public-api/runs/) |
| レポートをエクスポート | [Reports API](/ja/guides/core/reports/clone-and-export-reports/) |
| Artifacts をエクスポート | [Explore artifact graphs](/ja/guides/core/artifacts/explore-and-traverse-an-artifact-graph), [Download and use artifacts](/ja/guides/core/artifacts/download-and-use-an-artifact/#download-and-use-an-artifact-stored-on-wb) |

専用クラウドで保管されている Artifacts を [Secure Storage Connector](/ja/guides/models/app/settings-page/teams/#secure-storage-connector) で管理している場合、W&B SDK APIを使用してアーティファクトをエクスポートする必要はないかもしれません。

<Note>
大量の runs や Artifacts などがある場合、W&B SDK APIを使用してすべてのデータをエクスポートするのは遅くなる可能性があります。W&Bは、専用クラウドインスタンスを圧迫しないように、適切なサイズのバッチでエクスポートプロセスを実行することを推奨しています。
</Note>
