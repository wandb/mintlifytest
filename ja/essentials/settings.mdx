---
- title: グローバル設定
- description: Mintlifyでは、docs.jsonファイルを使用してドキュメントの外観と操作感を完全に制御できます
- icon: gear
---

すべての Mintlify サイトには`docs.json`コア設定を含むファイルが必要です。以下の[properties](#properties)について詳しく学びましょう。

## プロパティ

<ResponseField name="name" type="string" required>
  プロジェクトの名前。グローバルタイトルとして使用されます。

  Example: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  そのグループ内のすべてのページを含むグループの配列

  <Expandable title="ナビゲーション">
    <ResponseField name="group" type="string">
      グループの名前。

      Example: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      ページとして機能するマークダウンファイルへの相対パス。

      Example: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  ロゴ画像へのパス、または「light」モードと「dark」モードのロゴ画像へのパスを持つオブジェクト

  <Expandable title="ロゴ">
    <ResponseField name="light" type="string">
      ライトモードでのロゴへのパス
    </ResponseField>

    <ResponseField name="dark" type="string">
      ダークモードでのロゴへのパス
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      ロゴをクリックした時のリンク先
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  ファビコン画像へのパス
</ResponseField>

<ResponseField name="colors" type="Colors">
  グローバルテーマのHexカラーコード

  <Expandable title="カラー">
    <ResponseField name="primary" type="string" required>
      プライマリカラー。ライトモードでは、ハイライトされたコンテンツ、セクションヘッダー、アクセントなどに最もよく使用されます
    </ResponseField>

    <ResponseField name="light" type="string">
      ダークモード用のプライマリカラー。ダークモードでは、ハイライトされたコンテンツ、セクションヘッダー、アクセントなどに最もよく使用されます
    </ResponseField>

    <ResponseField name="dark" type="string">
      重要なボタンのプライマリカラー
    </ResponseField>

    <ResponseField name="background" type="object">
      ライトモードとダークモードの両方での背景色

      <Expandable title="オブジェクト">
        <ResponseField name="light" type="string" required>
          ライトモードでの背景のHexカラーコード
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモードでの背景のHexカラーコード
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  配列の`name`と`url`トップバーに含めたいリンク

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      ボタンの名前。

      Example: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      ボタンをクリックした時のURL。例：`https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="トップバーのコールトゥアクション">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      リンクはボタンとして表示されます。GitHubは提供されたURLのリポジトリ情報（GitHubスターの数を含む）を表示します。
    </ResponseField>

    <ResponseField name="url" type="string">
      もし`link`の場合：ボタンのリンク先。

      もし`github`の場合：GitHub情報を読み込むリポジトリへのリンク。
    </ResponseField>

    <ResponseField name="name" type="string">
      ボタン内のテキスト。`type`が`link`の場合のみ必要です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  バージョン名の配列。ナビゲーションバーにドロップダウンで異なるバージョンのドキュメントを表示したい場合にのみ使用します。
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  アンカーの配列、`icon`、`color`、および`url`を含みます。

  <Expandable title="アンカー">
    <ResponseField name="icon" type="string">
      アンカーを特徴付けるために使用される[Font Awesome](https://fontawesome.com/search?q=heart)アイコン。

      Example: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      アンカーラベルの名前。

      Example: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      アンカーに含まれるページを示すURLの先頭部分。一般的に、これはページを配置したフォルダの名前です。
    </ResponseField>

    <ResponseField name="color" type="string">
      アンカーアイコンの背景のHexカラー。`from`と`to`のプロパティを持つオブジェクトを渡すと、グラデーションにすることもできます。それぞれがHexカラーです。
    </ResponseField>

    <ResponseField name="version" type="string">
      正しいドキュメントバージョンが選択されるまでアンカーを非表示にしたい場合に使用します。
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      直接リンクでそのアンカー内のドキュメントに誰かを案内するまでアンカーを非表示にしたい場合は`true`を渡します。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか：「brands」、「duotone」、「light」、「sharp-solid」、「solid」、または「thin」
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  最上位のアンカーのデフォルト設定をオーバーライドします。

  <Expandable title="オブジェクト">
    <ResponseField name="name" default="Documentation" type="string">
      最上位のアンカーの名前
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Font Awesomeアイコン。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか：「brands」、「duotone」、「light」、「sharp-solid」、「solid」、または「thin」
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  ナビゲーションタブの配列。

  <Expandable title="タブ">
    <ResponseField name="name" type="string">
      タブラベルの名前。
    </ResponseField>

    <ResponseField name="url" type="string">
      タブに含まれるページを示すURLの先頭部分。一般的に、これはページを配置したフォルダの名前です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API設定の構成。

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      すべてのAPIエンドポイントのベースURL。`baseUrl`が配列の場合、ユーザーが切り替えられる複数のベースURLオプションが有効になります。
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="認証">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          すべてのAPIエンドポイントに使用される認証戦略。
        </ResponseField>

        <ResponseField name="name" type="string">
          APIプレイグラウンドで使用される認証パラメータの名前。

          メソッドが`basic`の場合、フォーマットは`[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          認証入力フィールドのプレフィックスとして設計されたデフォルト値。

          例えば、`inputPrefix`が`AuthKey`の場合、認証フィールドのデフォルト入力結果は`AuthKey`となります。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      APIプレイグラウンドの設定

      <Expandable title="プレイグラウンド">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          プレイグラウンドが表示されているか、非表示か、またはユーザーの対話性なしでエンドポイントのみを表示しているか`simple`

          APIプレイグラウンドの設定を構成する
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      このフラグを有効にすると、OpenAPIページのキーの順序がOpenAPIファイルで定義されたキーの順序と一致することが保証されます。

      <Warning>この動作はまもなくデフォルトで有効になり、その時点でこのフィールドは非推奨になります。</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPIファイルを指す文字列または文字列の配列のURL(複数可)または相対パス(複数可)。

  Examples:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  キー：プロパティのペアがソーシャルメディアプラットフォームとアカウントURLを表すソーシャルメディアアカウントのオブジェクト。

  Example:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      次の値のいずれか`website`、`facebook`、`x`、`discord`、`slack`、`github`、`linkedin`、`instagram`、`hacker-news`

      Example: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      ソーシャルプラットフォームへのURL。

      Example: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  フィードバックボタンを有効にする設定

  <Expandable title="フィードバック">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      ユーザーがプルリクエストを通じて編集を提案できるボタンを有効にします
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      ユーザーがドキュメントに関する問題を提起できるボタンを有効にします
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  ダークモードの切り替えをカスタマイズします。

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      新しいユーザーに常にライトモードまたはダークモードを表示したい場合に設定します。設定されていない場合、デフォルトではユーザーのオペレーティングシステムと同じモードになります。
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      ダーク/ライトモードの切り替えを非表示にするには true に設定します。`isHidden`と`default`を組み合わせて、ドキュメントを強制的にライトモードまたはダークモードのみで使用することができます。例えば：

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  すべてのページの背後に表示される背景画像。例を参照：[Infisical](https://infisical.com/docs)と[FRPC](https://frpc.io)。
</ResponseField>
